% ABSTRACT
% ============================================================
\begin{abstract}
Current AI computer use agents interact with graphical user interfaces statelessly---every click is treated as a first-time click, with no benefit from prior interactions. This contrasts sharply with human motor behavior, where repeated actions become faster and more automatic through practice. We present a spatiotemporal motor memory cache that enables autonomous GUI agents to develop ``muscle memory.'' The system integrates eight complementary mechanisms: R-Tree spatial indexing, Power Law of Practice speed-up, Fitts's Law movement timing, WindMouse trajectory adaptation, temporal decay, EMA position smoothing, action sequence prediction, and cross-session persistence. A three-layer resolution hierarchy---model-provided hints, accessibility API detection, and percentage-bucketed window-relative coordinates---ensures that \emph{every} mouse interaction builds cache entries, regardless of application or platform. Testing on real desktop applications across four operating systems demonstrates monotonically growing cache hit counts across sessions and across independent agent instances, with movement durations converging to 30\% of initial values for frequently-used targets. To our knowledge, this is the first system to integrate cognitive science motor learning models into the action-execution layer of AI GUI agents.
\end{abstract}

\begin{IEEEkeywords}
computer use agents, muscle memory, motor learning, Fitts's Law, Power Law of Practice, spatial caching, GUI automation
\end{IEEEkeywords}

% I. INTRODUCTION
% ============================================================
\section{Introduction}

A human expert's 100th click on ``Save'' is fast and automatic---a ballistic movement requiring no conscious attention. An AI computer use agent's 100th click on the same button takes exactly as long as its first. This asymmetry reveals a fundamental gap in current autonomous GUI systems: they have no concept of motor learning.

Recent advances in large language models (LLMs) have enabled a new class of \emph{computer use agents} (CUAs) that interact with desktop applications through screenshots and mouse/keyboard input~\cite{anthropic2024computer,openai2025operator,zhang2024ufo}. These systems have demonstrated impressive reasoning capabilities---deciding \emph{what} to click based on visual understanding. However, the execution layer---\emph{how} to reach the target---remains entirely stateless. Every mouse movement is computed from scratch, with no benefit from prior interactions with the same UI elements.

This stands in stark contrast to human motor behavior. Decades of cognitive science research have established that repeated aimed movements follow the \emph{Power Law of Practice}~\cite{newell1981}: movement time decreases as a power function of repetition count. Additionally, the base timing of aimed movements obeys \emph{Fitts's Law}~\cite{fitts1954}, and unused motor memories decay following patterns consistent with the \emph{Ebbinghaus forgetting curve}~\cite{ebbinghaus1885}.

We present a \emph{spatiotemporal motor memory cache} that bridges this gap. Our system sits entirely in the execution layer, orthogonal to and composable with any LLM backbone. It integrates eight complementary mechanisms:

\begin{enumerate}[noitemsep,topsep=0pt]
    \item \textbf{R-Tree spatial indexing} for proximity-based element lookup
    \item \textbf{Power Law of Practice} speed-up ($T(n) = T(1) \cdot n^{-0.4}$)
    \item \textbf{Fitts's Law} base movement timing
    \item \textbf{WindMouse trajectory adaptation} (gravity, wind, velocity)
    \item \textbf{Temporal decay} (7-day half-life) for cache freshness
    \item \textbf{EMA position smoothing} ($\alpha{=}0.3$) for element drift
    \item \textbf{Sequence prediction} for anticipatory targeting
    \item \textbf{Cross-session SQLite persistence}
\end{enumerate}

A three-layer resolution hierarchy ensures universal coverage: model-provided semantic hints (Layer~3), accessibility API auto-detection (Layer~2), and percentage-bucketed window-relative coordinates (Layer~1). Layer~1 requires only the foreground window's bounding rectangle---available on every major OS---ensuring that \emph{every} click builds muscle memory, even in applications with no accessibility support.

\section{Background and Related Work}

\subsection{Cognitive Science Foundations}

\textbf{Power Law of Practice.} Newell and Rosenbloom~\cite{newell1981} established that the time to perform a task decreases as a power function of the number of practice trials: $T(n) = T(1) \cdot n^{-\alpha}$, where $\alpha$ typically ranges from 0.2 to 0.6 for motor tasks. This law has been observed across diverse domains including cigar rolling, card sorting, and typing.

\textbf{Fitts's Law.} Fitts~\cite{fitts1954} showed that the time required for an aimed movement is logarithmically related to the ratio of distance to target width: $MT = a + b \cdot \log_2(D/W + 1)$. This remains the fundamental model of aimed movements in HCI~\cite{mackenzie1992,soukoreff2004}.

\textbf{Ebbinghaus Forgetting Curve.} Ebbinghaus~\cite{ebbinghaus1885} demonstrated that memory strength decays exponentially with time. We implement this as a half-life model where cache entry confidence halves every 7~days.

\textbf{Motor Memory Consolidation.} Willingham~\cite{willingham1998} described how procedural motor memories become automatic with repetition, transitioning from explicit to implicit processing. Our system models this transition through the WindMouse parameter adaptation: as hit count grows, cursor trajectories become straighter and faster---mimicking the shift from conscious to automatic motor control.

\subsection{AI GUI Agent Systems}

Current computer use agents operate statelessly at the execution layer. Anthropic's Claude Computer Use~\cite{anthropic2024computer} processes screenshots and outputs coordinate-based click commands, computing each target position from scratch via LLM inference. OpenAI's Operator~\cite{openai2025operator} follows a similar architecture. Microsoft's UFO~\cite{zhang2024ufo} adds multi-agent coordination but maintains stateless execution. WebVoyager~\cite{he2024webvoyager} and SeeAct~\cite{zheng2024seeact} focus on web navigation with no motor adaptation. None of these systems maintain any memory of prior interactions at the motor level.

\subsection{Motor Simulation and Caching Approaches}

\textbf{Muscle-Mem}~\cite{musclemem2024} caches entire tool-calling sequences for deterministic replay, operating at the task abstraction layer (``what actions to take'') rather than the motor execution layer (``how to reach the target''). It contains no spatial indexing, no gradual speed-up, and no movement adaptation.

\textbf{Ghost-Cursor}~\cite{ghostcursor} implements static Fitts's Law simulation for browser testing with B\'{e}zier curves. Every movement to the same target takes identical time---there is no learning or cross-session persistence.

\subsection{Three-Layer Cache Resolution}

Every mouse action must resolve to a cache key---a combination of \emph{application name} and \emph{element hint}. We employ a three-layer hierarchy to ensure universal coverage:

\textbf{Layer~3 (Model-provided hint).} When the LLM provides a semantic label such as \texttt{element\_hint="Save button"}, this becomes the cache key directly. This is the highest-precision layer but depends on LLM cooperation.

\textbf{Layer~2 (Accessibility API).} When no hint is provided, the engine queries the operating system's accessibility tree for the UI element at the click coordinates, producing keys like \texttt{"button:Save"}. Available for native applications on Windows (UI Automation), Linux (AT-SPI2), and macOS (Accessibility API).

\textbf{Layer~1 (Percentage-bucketed coordinates).} The universal fallback. Requires only the foreground window's bounding rectangle. Click coordinates are converted to window-relative percentages and bucketed at 3\% resolution (${\sim}33$ buckets per axis), producing synthetic hints like \texttt{"@24\%,18\%"}. This layer:

\begin{itemize}[noitemsep,topsep=0pt]
    \item Survives \textbf{window moves} (window-relative coordinates)
    \item Survives \textbf{window resizes} (percentage-based)
    \item Works on \textbf{any application}, including games, Electron apps, and custom UIs with no accessibility support
\end{itemize}

The application name is obtained from the foreground window's process (e.g., \texttt{"notepad.exe"}, \texttt{"firefox"}). Remote desktop shells (RDP, VNC, terminal emulators) are detected and excluded via a passthrough list---their foreground window is a container, not the actual application being controlled.

\subsection{Three-Tier Cache Lookup}

Given a resolved key \texttt{(app\_name, hint)}, the cache performs a tiered lookup:

\textbf{Tier~1 (Hot cache).} An in-memory Python dictionary keyed by \texttt{"app|hint"} with $O(1)$ exact-match lookup. Sub-microsecond latency. Loaded from SQLite at startup.

\textbf{Tier~2 (R-Tree spatial proximity).} A SQLite R-Tree virtual table~\cite{guttman1984rtree,sqlite_rtree} enables proximity search within a 50-pixel radius, scoped to the same application. Used when Tier~1 misses but coordinates overlap with a known target (e.g., a button that has moved slightly). Latency ${\sim}0.1$ms.

\textbf{Tier~3 (Cache miss).} Returns \texttt{hit\_count=0}. Movement uses default WindMouse parameters.

\subsection{Data Model}

The persistent store uses SQLite with Write-Ahead Logging (WAL) for concurrent read/write safety:

\begin{itemize}[noitemsep,topsep=0pt]
    \item \texttt{memories}: Primary table storing \texttt{app\_name}, \texttt{element\_hint}, EMA-smoothed position $(x, y, w, h)$, \texttt{hit\_count}, \texttt{miss\_count}, timestamps, \texttt{confidence}, and \texttt{prev\_hint} for sequence tracking.
    \item \texttt{memories\_rtree}: R-Tree spatial index on element bounding boxes.
    \item \texttt{sequences}: Action pair tracking (from $\rightarrow$ to) with frequency counts for workflow prediction.
\end{itemize}

An LRU eviction policy caps the cache at 10{,}000 entries, deleting the oldest 10\% by \texttt{last\_hit\_ts} when capacity is reached.

\subsection{Cross-Platform Support}

The cache algorithm is platform-agnostic (pure Python, stdlib \texttt{sqlite3}). Platform-specific operations---foreground window queries, cursor control, accessibility API access---are delegated to backend modules:

\begin{itemize}[noitemsep,topsep=0pt]
    \item \textbf{Windows:} Win32 \texttt{SendInput} via ctypes; \texttt{GetForegroundWindow} for window info
    \item \textbf{WSL2:} TCP bridge daemon on Windows (port 19542) relays Win32 calls
    \item \textbf{Linux/X11:} \texttt{xdotool} for window queries; \texttt{/proc/\{pid\}/comm} for process name
    \item \textbf{macOS:} AppleScript via \texttt{osascript} for window info; \texttt{cliclick} for input
\end{itemize}

Foreground window queries are TTL-cached (50ms in the engine, 100ms on subprocess-based platforms) to amortize overhead.

% IV. MOTOR ADAPTATION MODEL
% ============================================================
\section{Motor Adaptation Model}

This section details the six mechanisms by which repeated interactions produce faster, more efficient cursor movements.

\subsection{Power Law of Practice}

Following Newell and Rosenbloom~\cite{newell1981}, we model the speed-up of repeated aimed movements as:
\begin{equation}
    T(n) = T_{\text{base}} \cdot n^{-\alpha}, \quad \text{floored at } T_{\text{base}} \cdot \beta
    \label{eq:power_law}
\end{equation}
where $n$ is the hit count (number of prior interactions with the same target), $\alpha = 0.4$ is the learning rate (mid-range for motor tasks), and $\beta = 0.3$ is the asymptotic floor (the target can never be reached faster than 30\% of the initial duration).

Fig.~\ref{fig:power_law} shows the adaptation curve. At $n{=}10$, movement duration has dropped to approximately 40\% of initial. By $n{=}64$, it reaches the 30\% floor, modeling the asymptotic performance observed in human motor skill studies.

\begin{figure}[t]
    \centering
    \includegraphics[width=\columnwidth]{figures/fig2_power_law.pdf}
    \caption{Power Law of Practice curve. Movement duration decreases as $n^{-0.4}$ with a 30\% floor. Key points: $n{=}10$ yields ${\sim}40\%$ of initial duration; $n{=}64$ reaches the asymptotic floor.}
    \label{fig:power_law}
\end{figure}

\subsection{Fitts's Law Base Timing}

The initial movement duration $T_{\text{base}}$ is computed using the Shannon formulation of Fitts's Law~\cite{fitts1954,mackenzie1992}:
\begin{equation}
    MT = a + b \cdot \log_2\!\left(\frac{D}{W} + 1\right)
    \label{eq:fitts}
\end{equation}
with $a = 0.05$\,s (base reaction time), $b = 0.11$\,s (movement time coefficient), and a floor of 0.07\,s. Gaussian jitter ($\sigma_a = 0.01$, $\sigma_b = 0.015$) is added to both coefficients to produce human-like variability. The Power Law factor from Eq.~\ref{eq:power_law} is then applied to the resulting duration.

\subsection{WindMouse Trajectory Adaptation}

Cursor trajectories are generated using the WindMouse model, which simulates a particle under two forces: \emph{gravity} (directed toward the target) and \emph{wind} (random perturbation). The model operates in two phases:

\begin{itemize}[noitemsep,topsep=0pt]
    \item \textbf{Ballistic phase} ($d \geq d_h$): Active wind perturbation produces curved, exploratory paths.
    \item \textbf{Homing phase} ($d < d_h$): Wind decays exponentially; gravity dominates, producing precise final approach.
\end{itemize}

where $d$ is the remaining distance and $d_h = 12$\,px is the homing threshold.

As practice count increases, WindMouse parameters adapt via logarithmic scaling (saturating at ${\sim}64$ hits):
\begin{align}
    f &= \min\!\left(1.0,\; \frac{\log_2(n)}{6.0}\right) \label{eq:factor} \\
    G(n) &= 9.0 + 11.0 \cdot f  &&\text{(gravity: 9 $\to$ 20)} \label{eq:gravity} \\
    W(n) &= 3.0 \cdot (1 - 0.7 \cdot f) &&\text{(wind: 3 $\to$ 0.9)} \label{eq:wind} \\
    V(n) &= 20.0 + 15.0 \cdot f &&\text{(max vel: 20 $\to$ 35)} \label{eq:maxvel}
\end{align}

The net effect is that practiced trajectories become straighter (higher gravity), less wobbly (lower wind), and faster (higher maximum velocity)---closely paralleling the transition from conscious to automatic motor control observed in humans~\cite{willingham1998}. Fig.~\ref{fig:trajectory} illustrates this adaptation.

\begin{figure}[t]
    \centering
    \includegraphics[width=\columnwidth]{figures/fig3_trajectory.pdf}
    \caption{WindMouse cursor trajectories. (a)~First interaction ($n{=}1$): wobbly, exploratory path. (b)~After 30 repetitions: nearly straight, confident path. Both share the same start and end points.}
    \label{fig:trajectory}
\end{figure}

\subsection{Temporal Decay}

Cache entries lose confidence over time, modeling the Ebbinghaus forgetting curve~\cite{ebbinghaus1885}:
\begin{equation}
    c(t) = c_0 \cdot 2^{-t / \tau}
    \label{eq:decay}
\end{equation}
where $\tau = 604{,}800$\,s (7~days) is the half-life. Entries with $c(t) < 0.1$ are treated as forgotten (cache miss). Each successful interaction resets $c_0 = 1.0$. Additionally, each \emph{miss} (target not found where expected) penalizes confidence by $-0.3$, providing rapid adaptation when UI layouts change.

\subsection{EMA Position Smoothing}

UI elements may drift between sessions due to window resizing, layout changes, or dynamic content. We track element positions using an Exponential Moving Average:
\begin{equation}
    \hat{x}_{n+1} = \hat{x}_n + \alpha \cdot (x_{\text{obs}} - \hat{x}_n)
    \label{eq:ema}
\end{equation}
with $\alpha = 0.3$. This smoothing absorbs drift while preserving the learned position of stable elements. The same EMA is applied to element width and height.

\subsection{Sequence Prediction}

The system tracks action transitions as ordered pairs: when the user clicks target $B$ after target $A$, the pair $(A \to B)$ is recorded with a frequency count. The \texttt{predict\_next(app, current\_hint)} function returns the most frequent successor, enabling anticipatory pre-computation of the next target's cache entry. Example from live usage: after the sequence ``WhatsApp taskbar icon $\to$ Andr\'{e}s P Scale chat'' occurs twice, the system predicts this transition with high confidence.

\section{Implementation}

The system is implemented in Python with zero external dependencies for the core cache (\texttt{sqlite3} is part of the standard library). Key implementation details:

\textbf{Storage.} SQLite with Write-Ahead Logging (WAL) mode and \texttt{NORMAL} synchronous setting balances durability and performance. The R-Tree virtual table is compiled into SQLite by default on all major platforms.

\textbf{Integration.} The cache is exposed to LLMs via the Model Context Protocol (MCP)~\cite{mcp2024}, which provides an \texttt{element\_hint} parameter on all mouse action tools (\texttt{click}, \texttt{double\_click}, \texttt{right\_click}, \texttt{move\_mouse}). The engine transparently performs cache resolution, lookup, and recording around each action.

\textbf{Scale.} The implementation comprises approximately 460 lines for the cache module, 100 lines for engine integration, and 200 lines for motor adaptation in the platform executor. A test suite of 154 unit tests covers cache CRUD operations, spatial proximity, temporal decay, power law adaptation, sequence tracking, persistence, eviction, three-layer resolution, passthrough app exclusion, and percentage-bucket alignment.

\subsection{Qualitative Observations}

\textbf{Cross-agent persistence.} The most notable result is that Agent~B benefits from Agent~A's prior interactions without any explicit knowledge transfer. The shared SQLite database acts as organizational memory---analogous to how experienced team members benefit from a shared workspace layout.

\textbf{Resize resilience.} Percentage-based coordinate bucketing (Layer~1) correctly resolves cache hits after window resizing. A target at 25\% of an 800\,px window (200\,px) maps to the same bucket as 25\% of a 1200\,px window (300\,px).

\textbf{Passthrough exclusion.} Remote desktop shells (RDP, VNC) are correctly excluded from window-based caching, preventing cache entries from being associated with a container application rather than the actual UI being controlled.

\subsection{Limitations}

The current evaluation has several limitations. First, we have not conducted a formal controlled study comparing movement time and trajectory quality with and without the motor memory cache. Second, the Power Law exponent ($\alpha = 0.4$) and floor ($\beta = 0.3$) are fixed values from the literature rather than fitted to observed agent behavior. Third, the evaluation is single-user on a limited set of applications. Fourth, cursor trajectory ``naturalness'' is inherently subjective without a formal human-likeness metric.

% VII. DISCUSSION
% ============================================================
\section{Discussion}

\textbf{Orthogonality.} The motor memory cache operates entirely in the execution layer. It is compatible with any LLM backbone (Claude, GPT-4, open-source models) and any decision framework. Importantly, it does not change \emph{what} the agent clicks---only \emph{how fast and how directly} the cursor reaches the target. This makes it a universal enhancement rather than a competing approach.

\textbf{Emergent trajectory convergence.} As hit count grows, the WindMouse parameter adaptation produces trajectories that converge on near-straight-line paths with smooth velocity profiles. This resembles the minimum-jerk trajectory~\cite{soukoreff2004} characteristic of expert human pointing---an emergent property of the parameter adaptation, not explicitly programmed.

\textbf{Shared memory across agents.} Because the cache is stored in SQLite on disk, multiple independent agent instances share the same motor memory. Agent~B's cursor movements are more efficient for targets that Agent~A has previously interacted with. This models organizational knowledge transfer: a new team member benefits from the team's collective familiarity with a codebase.

\textbf{Living system.} The parameters reported in this paper ($\alpha = 0.4$, $\beta = 0.3$, $\tau = 7$~days, EMA $\alpha = 0.3$, spatial radius = 50~px) are all configurable. As more empirical data is collected, these values may be tuned or made adaptive.

% VIII. CONCLUSION
% ============================================================
\section{Conclusion and Future Work}

We have presented the first system to integrate cognitive science motor learning models---Power Law of Practice, Fitts's Law, Ebbinghaus temporal decay---into the action-execution layer of autonomous GUI agents. The spatiotemporal motor memory cache, with its three-layer resolution hierarchy and eight complementary mechanisms, enables any computer use agent to develop muscle memory that persists across sessions and transfers across agent instances.

Future work includes: (1)~formal A/B studies comparing movement time and trajectory quality with and without the cache; (2)~adaptive $\alpha$ fitting that learns the power law exponent from observed data for each element type; (3)~anticipatory cursor movement that begins moving toward the predicted next target before the LLM completes its decision; (4)~a shared cloud cache for fleet-wide motor memory across users and machines; and (5)~integration with richer accessibility tree information for improved Layer~2 resolution.
